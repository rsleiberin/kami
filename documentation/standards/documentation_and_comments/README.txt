# Documentation and Comments

## Introduction
This directory provides guidelines for writing clear, effective documentation and comments within the Project Kami codebase. Adhering to these standards ensures that both human developers and AGI modules can understand and maintain the code.

## For Humans
- Follow the guidelines for commenting your code.
- Make use of documentation tools like Doxygen for generating code documentation.

## For AGI
- Utilize the protocols for embedding documentation within the code.
- Use standardized comment tags to assist in automated documentation generation.

## How to Contribute
To contribute to these guidelines, please refer to the higher-level `standards/README.txt` for the procedure to update or add new content.
